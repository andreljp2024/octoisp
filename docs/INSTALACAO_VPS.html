OctoISP · Instalação e Deploy em VPS

Um passo a passo ilustrado para subir o OctoISP em produção sem complicação, com TLS automático, validação de conexões e assistente web.

Pré‑requisitos

  • VPS Linux (Ubuntu/Debian)
  • Domínio apontado para o IP da VPS
  • Portas 80 e 443 liberadas
  • Docker + Docker Compose

Fluxo de deploy

Passo a passo rápido

1 Preparar VPS
sudo apt update && sudo apt upgrade -y
sudo apt install -y git curl ufw
sudo ufw allow OpenSSH
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
2 Instalar Docker
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
newgrp docker
3 Clonar e preparar env
git clone <seu-repo> octoisp
cd octoisp
cp .env.production.example .env.production
4 Subir stack base
docker compose -f docker-compose.prod.yml --env-file .env.production up -d
5 Abrir o assistente web
https://seu-dominio/setup
No assistente você configura Supabase, bancos, TLS (Let’s Encrypt) e dispara o deploy com logs em tempo real.

Checklist de configuração

  • FRONTEND_URL e ALLOWED_ORIGINS preenchidos
  • SUPABASE_URL + ANON_KEY + JWT_SECRET
  • DATABASE_URL, METRICS_DB_URL, REDIS_URL
  • SETUP_WIZARD_ENABLED e TOKEN
  • SETUP_HOST_DIR com caminho absoluto

Boas práticas

Para produção pública, use TLS válido (Let’s Encrypt ou certificado oficial).
Após o deploy, desative o assistente (`SETUP_WIZARD_ENABLED=false`).

Renovação automática do Let’s Encrypt

Ative o container de renovação:

docker compose -f docker-compose.prod.yml --env-file .env.production --profile optional-services up -d certbot

Checagem rápida

docker compose -f docker-compose.prod.yml ps
curl -k https://seu-dominio/health
